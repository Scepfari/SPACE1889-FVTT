Version 2.0.5 -
Time of the three nameless empires

 

v2.0.5

  • Dice dialog improvements: 
    • for primary attribute rolls: Toggle whether to use single or double primary attribute value, display of resulting dice count, as well as tooltip for dice count in chat explaining about composition
    • The dialogs now have the option whether the roll should be done publicly, privately or privately & SL
  • New setting option: The player can now choose to display the modification dialog for every attribute roll, regardless of whether the [SHIFT] or [CTRL] keys were pressed.
  • Bugfix: missing damage output for counterattacks like "Counterstrike" and "Riposte".

 

v2.0.4

  • New customizable key combination for closing the active window. The default assignment is the ESC key, which prevents Foundry's usual behavior of closing all open windows, including minimized ones, when the ESC key is pressed. If you want the ESC key to work as it used to, you need to assign a different key combination to the function to close the active window.
    Thanks to Forien on Discord for sharing the source code.

 

v2.0.3

  • New option in the settings to treat NPCs like PCs. When active, NPCs are not defeated at 0 health, but die at -5 health, just like characters.
  • Weapons can now cause special effects that are applied additionally or exclusively on hitting. An example of this is the flamethrower now included in the compendium.
  • New ammunition type chemicals and associated capacity type tank.
  • New external link to the armory, available via the Foudry icon in the upper left corner
       
  • Fix: Attacks with area damage weapons are now only successful when they exceed the defense test, not already at a tie.
  • Fix: Auto Defense Area Damage: Deductions/surcharges are now calculated correctly, giving large creatures less area damage and small creatures more. Previously it was incorrectly the other way around.
  • Fix: V11 the "dead marker" automatically generated by damage now also covers the entire token, as with V10

 

v2.0.2

  • New option in the settings to define the indentation depth of the folders and subfolders

  • List of suggested modules, available via the Foundry icon in the top left corner

  • Bugfix: Negative health on creatures is now applied correctly as it is on PCs and NPCs.
  • Bugfix: Negative health no longer leads to negative perception/initiative values
  • Bugfix: V11 Warning regarding pack.private removed by moving to pack.ownership
  • Bugfix: V11 Immersion frame of the hotbar

 

v2.0.1

  • The sheets of characters, NPCs, creatures and vehicles now have two new text fields:

    • GM Info: Is visible only to the game master, even if the players are owners of the actor.
    • Player Info: Is visible and editable for players even if they have limited rights to the actor. However, editing with limited rights only works if the game master is logged into the game.
  • Fix: The effect paralyzed now also affects the values in the character sheet, in that the active defense drops to 0 and this is included in all derived values. Paralyzed also means that full defense is no longer possible.
  • Fix: Negative health now automatically affects the secondary and parts of the primary (strength and dexterity) according to the rules. This also includes the special defensive maneuvers like Parry and co.
  • Fix: Effects in Foundry V11

 

v2.0.0

  • Foundry VTT V10 and V11 support 
  • Introduction of the new item type "Container / Storage location". This replaces the old body/backpack/storage assignment, relocating items is still possible by clicking on the displayed location, or by drag & drop. The order of the containers in the character sheet can also be changed via drag & drop onto other containers.
  • All actors are automatically converted when switching to this version. If you import actors from older versions or from compendiums you created yourself, everything that was previously in the storage or backpack will first end up on the body. If you drag these actors into a scene and select them, you can trigger the conversion manually using the following chat command:
    /addContainer
  • The actor's current load is now calculated including the containers carried, and the Maluses from overloading flows directly into the attributes, and thus into all rolls.  
  • Weapons can now only be drawn or reloaded if they are carried by the player. Exceptions are of course the guns, which are much too big and heavy to be carried.
  • Space menu  Ready the weapons:  
    • Function can now also be applied to selected characters only
    • Function is additionally accessible via a key combination [K] 
  • The system now automatically calculates the deductions for rolls with negative health
  • The sheets of characters, NPCs and creatures now contain links ( ) that can be used to directly open the corresponding compendium.
  • Fix unconscious at talent Diehard
